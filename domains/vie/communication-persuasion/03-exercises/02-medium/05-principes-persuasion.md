# Exercices (medium) — Module 05 : Les principes de persuasion

> **Prerequis** : avoir lu `01-theory/05-principes-persuasion.md` et fait les exercices `01-easy/05-principes-persuasion.md`
> **Duree estimee** : 40-50 min
> **Solutions** : `03-exercises/solutions/05-principes-persuasion-medium.md`

> **Garde-fou ethique (charte CTR : Consentement / Transparence / Reversibilite)** : on apprend les principes de Cialdini d'abord pour s'en defendre, ensuite pour les utiliser honnetement. Un principe qui ne passe pas le test CTR est une manipulation, intentionnelle ou non. Aucune fausse rarete, aucun faux temoignage, aucune fausse autorite.

---

## Exercice 1 — Combiner deux principes honnetement (et reperer la sur-combinaison)

### Objectif

Apprendre qu'un principe seul, bien utilise, est souvent plus ethique et plus credible que l'empilement de plusieurs leviers — qui signale la manipulation (Module 05 : "un message qui exploite tous les principes a la fois... peut etre recu comme une manipulation").

### Consigne

**Contexte** : vous organisez un atelier gratuit de premiers secours pour les habitants de votre immeuble. Vous voulez que les gens s'inscrivent.

1. **Identifiez** les 2 principes de Cialdini les plus naturellement *honnetes* dans ce contexte (parmi : reciprocite, preuve sociale, autorite, rarete, sympathie, coherence, unite). Justifiez pourquoi ils sont legitimes ici (rarete reelle ? autorite pertinente ?).

2. **Redigez** un message d'invitation qui mobilise ces 2 principes de facon honnete et passe le test CTR.

3. **Contre-exemple** : redigez la *version sur-combinee manipulatoire* du meme message (qui empile 4-5 principes avec fausse urgence, fausse preuve sociale, etc.) — **puis annotez chaque procede problematique**. Le but est de voir le contraste, pas de l'utiliser.

4. **Comparez** : laquelle inspire le plus confiance ? Pourquoi l'empilement se retourne-t-il contre l'emetteur ?

### Criteres de reussite

- [ ] Les 2 principes choisis sont pertinents et leur legitimite est justifiee (rarete reelle, autorite pertinente, etc.).
- [ ] Le message honnete passe le test CTR explicitement.
- [ ] La version sur-combinee est annotee : chaque procede problematique est nomme (fausse urgence, preuve sociale invérifiable...).
- [ ] La comparaison explique pourquoi l'empilement detruit la credibilite.
- [ ] Aucun procede manipulatoire n'est presente comme recommandable.

## Exercice 2 — Reecrire une demande faible avec le bon principe

### Objectif

Choisir le principe *adapte au contexte et a la relation* (pas le plus "puissant"), et l'appliquer sans forcer.

### Consigne

Pour chacune des trois demandes faibles ci-dessous :
1. Diagnostiquez pourquoi elle est faible (aucun levier, ou levier inadapte).
2. Choisissez **un seul** principe de Cialdini adapte a CE contexte. Justifiez.
3. Reecrivez la demande honnetement.
4. Verifiez le test CTR (C/T/R en une ligne chacun).

**Demande 1** (a un voisin) : "Tu pourrais arroser mes plantes pendant mes vacances ?"

**Demande 2** (a une collegue, pour relire un dossier) : "Tu peux relire mon dossier ? C'est urgent."

**Demande 3** (a des membres d'un club de lecture, pour proposer un nouveau format) : "Je propose qu'on change de format de reunion."

### Criteres de reussite

- [ ] Le diagnostic de faiblesse est precis pour chaque demande.
- [ ] Le principe choisi est adapte a la relation (ex. reciprocite avec un voisin, unite dans un club).
- [ ] La reecriture integre le principe sans forcer ni fabriquer.
- [ ] Le test CTR est verifie pour les trois (C/T/R).
- [ ] Aucune fausse urgence ni fausse rarete n'est utilisee (la demande 2 ne doit pas inventer une urgence qui n'existe pas).

## Exercice 3 — Construire sa defense : transformer la checklist en reflexe

### Objectif

Passer de la reconnaissance passive des principes a une *defense active* : appliquer la checklist en 5 questions a des sollicitations reelles et formuler une reponse qui demande l'information manquante sans agressivite.

### Consigne

Pour chacune des trois sollicitations, appliquez la checklist de defense (5 questions du module), identifiez le(s) point(s) faible(s), et redigez une **reponse-type** qui (a) ne cede pas a la pression, (b) demande poliment l'information manquante, (c) laisse la porte ouverte si la sollicitation est honnete.

**Sollicitation 1** (telephone) : "Bonjour, vous avez ete selectionne parmi nos meilleurs clients pour une offre exclusive, mais il faut confirmer aujourd'hui car les places partent vite."

**Sollicitation 2** (collegue) : "Tu m'avais dit que tu etais pour ce projet, donc je compte sur toi pour le defendre en reunion demain." (vous aviez juste dit que l'idee etait "interessante")

**Sollicitation 3** (vendeur en magasin) : "Franchement, entre nous, je vous le dis comme a un ami : c'est le meilleur rapport qualite-prix, prenez-le les yeux fermes."

### Criteres de reussite

- [ ] La checklist en 5 questions est appliquee a chaque sollicitation (les questions pertinentes).
- [ ] Le levier active est nomme (rarete/urgence, piege de coherence, sympathie/fausse proximite).
- [ ] La reponse-type ne cede pas a la pression temporelle.
- [ ] La reponse demande l'information manquante (le module : "un interlocuteur honnete vous la donnera ; un manipulateur esquivera").
- [ ] La reponse reste courtoise et laisse une porte ouverte si la demarche est legitime.
