# Exercice Capstone (medium) — Convaincre honnetement : extensions du portfolio

> **Prerequis** : avoir realise le capstone de base `01-easy/07-capstone-convaincre-honnetement.md` (3 livrables : pitch, ecrit persuasif, simulation de negociation)
> **Duree estimee** : 70-90 min | **Mobilise** : Modules 01-06
> **Solutions** : `03-exercises/solutions/07-capstone-convaincre-honnetement-medium.md`

> **Garde-fou ethique (charte CTR)** : chaque livrable est evalue sur **clarte, ethique (filtre CTR) et resultat mutuel**. Le test final du domaine s'applique a tout : *"si l'autre savait exactement ce que j'essaie de faire, approuverait-il ma methode ?"*

---

## Contexte (suite du capstone)

Vous reprenez le scenario du capstone de base : l'association **Quartier Vivant** veut obtenir le local municipal du 12 rue des Acacias, face a Mme Laroche (adjointe a la vie associative) et a un concurrent (une start-up d'evenementiel). Les extensions ci-dessous *enrichissent le portfolio* : elles supposent que vos 3 livrables de base existent deja.

---

## Extension 1 — La preparation invisible : dossier d'ecoute et d'interets

### Objectif

Documenter le travail *amont* qui rend les 3 livrables credibles : ce qu'un bon communicant prepare avant de parler (Module 02 : "avant de parler, comprendre" ; Module 06 : interets vs positions).

### Consigne

Avant un pitch ou une negociation, on ne sait pas vraiment convaincre tant qu'on n'a pas cartographie l'autre. Produisez un **dossier de preparation** d'une page contenant :

1. **Carte des interets de Mme Laroche** : au-dela de sa position ("je dois choisir entre deux candidats"), 3-4 interets probables (politiques, budgetaires, de risque, d'image publique). Pour chacun, comment votre demande peut-elle y repondre ?

2. **Carte des interets du concurrent** (la start-up) : quels sont ses atouts et ses faiblesses *de son point de vue a elle* ? L'objectif n'est pas de la denigrer (interdit par CTR) mais de comprendre le choix de Mme Laroche.

3. **Vos 3 questions calibrees** : 3 questions ("Comment..." / "Qu'est-ce qui...") que vous poseriez a Mme Laroche en debut d'entretien pour decouvrir ses vrais criteres avant de derouler votre pitch.

4. **Anticipation d'objections** : listez 3 objections probables et, pour chacune, une reponse honnete (qui ne cache aucun cout reel pour la mairie).

### Criteres de reussite

- [ ] Au moins 3 interets de Mme Laroche identifies, distincts de sa position, avec reponse de votre demande.
- [ ] Les interets du concurrent sont analyses sans denigrement (CTR : respect).
- [ ] 3 questions calibrees, ouvertes et non orientees.
- [ ] 3 objections + reponses honnetes (sans dissimulation de cout).
- [ ] Le dossier reste factuel : aucune supposition presentee comme certitude.

## Extension 2 — Adapter le message a trois publics differents

### Objectif

Montrer la maitrise de l'adaptation (Module 03/04 : calibrer a l'etat de connaissance de l'audience), en declinant la meme demande pour trois interlocuteurs aux preoccupations differentes.

### Consigne

Le meme projet (obtenir le local) doit etre presente differemment selon le public. Produisez **trois versions courtes** (80-120 mots chacune) du message-cle :

**Version A — Mme Laroche** (decideuse, soucieuse du risque et de l'image publique) : registre institutionnel, met en avant la fiabilite, les comptes-rendus, le benefice pour la commune.

**Version B — les habitants du quartier** (a mobiliser pour soutenir la demande) : registre chaleureux, concret, SUCCESs, met en avant ce que ça change pour eux.

**Version C — les benevoles de l'association** (a remobiliser pour tenir l'engagement) : registre interne, unite, reconnaissance, et clarte sur ce qui est attendu d'eux.

Pour chaque version, indiquez en une ligne quelle(s) dimension(s) (ethos/pathos/logos et/ou SUCCESs) vous avez privilegiee(s) et pourquoi pour CE public.

### Criteres de reussite

- [ ] Les trois versions transmettent la meme demande de fond mais avec un angle adapte.
- [ ] La version A privilegie la fiabilite/le benefice institutionnel.
- [ ] La version B est concrete et chaleureuse (SUCCESs visible).
- [ ] La version C mobilise l'unite et clarifie l'attendu, sans culpabiliser.
- [ ] Chaque version reste honnete : les memes faits, presentes sous l'angle pertinent (pas des faits differents selon le public).
- [ ] Le choix d'angle est justifie pour chaque public.

## Extension 3 — Auto-evaluation a la grille du domaine

### Objectif

Boucler la boucle metacognitive : evaluer ses propres livrables avec la grille du Module 07 (clarte / ethique CTR / resultat mutuel) et identifier un axe d'amelioration concret.

### Consigne

Reprenez vos 3 livrables du capstone de base. Pour chacun, remplissez la grille d'evaluation du domaine :

| Livrable | Clarte (1-5) | Ethique CTR (1-5) | Resultat mutuel (1-5) | Point fort | Axe d'amelioration |
|----------|--------------|-------------------|------------------------|------------|--------------------|

Puis :
1. Identifiez votre **livrable le plus faible** et expliquez pourquoi.
2. Reecrivez-en **un passage** (3-5 phrases) pour corriger l'axe d'amelioration identifie.
3. En 3 lignes : quelle competence du domaine vous reste-t-il le plus a travailler ? (honnetete metacognitive — Module sur l'auto-evaluation).

### Criteres de reussite

- [ ] La grille est remplie pour les 3 livrables, avec notes justifiees (pas juste des chiffres).
- [ ] Le livrable le plus faible est identifie et l'analyse est honnete (pas complaisante).
- [ ] Le passage reecrit corrige reellement l'axe identifie.
- [ ] L'auto-evaluation finale nomme une competence a travailler de facon specifique.
- [ ] La grille verifie explicitement que chaque livrable passe le filtre CTR.
