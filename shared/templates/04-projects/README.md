# 04-projects — Mini-projets / capstones libres

Ce dossier accueille les **mini-projets libres** et les **capstones additionnels**
lies au domaine, en dehors du capstone fil-rouge (qui, lui, vit dans
`02-code/<dernier-module>`).

## Usage

- Un sous-dossier par projet : `01-<nom-projet>/`, `02-<nom-projet>/`, ...
- Chaque projet contient au minimum un `README.md` (objectif, consigne, criteres
  de reussite) et, si pertinent, le code de depart et/ou un corrige.
- Contrairement aux `05-projets-guides/`, ces projets ne sont **pas** contraints au
  contexte logistique partage : ils peuvent explorer n'importe quel angle du domaine.

> Supprime ce placeholder quand tu ajoutes ton premier projet.
