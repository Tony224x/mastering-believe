# 05-projets-guides — Projets reels appliques (contexte logistique)

Projets longs appliques au **contexte logistique automatise partage** du repo
(editeur fictif de simulation logistique). Ils montrent comment combiner les patterns
appris sur un cas concret de bout en bout. Le contexte metier unique est decrit dans
[`shared/logistics-context.md`](../../logistics-context.md).

## Structure

- Un sous-dossier numerote par projet : `01-<projet>/`, `02-<projet>/`, ...
- Chaque projet contient :
  - `README.md` — contexte metier, objectif technique, consigne, etapes guidees, criteres de reussite
  - `solution/` — corrige commente (le **why** des choix)

Voir `01-projet-exemple/` comme gabarit de depart, puis renomme-le et complete-le.

## Recommandation

Minimum 3 projets guides par domaine finalise, branches sur le fil rouge decrit dans
`shared/logistics-context.md`.
